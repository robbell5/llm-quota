<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code hook stdin → cache writer | Private local hook payload crosses into app-owned parser; malformed data must not corrupt the cache. |
| llm-quota installer → user Claude settings | App mutates user-owned `.claude/settings.json`; unrelated hooks must be preserved. |
| cache writer → `~/.cache/llm-quota/claude.json` | App writes local cache consumed later by `ClaudeReader`; partial writes must be avoided. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Tampering | `internal/install/claude_hook.go` config mutation | mitigate | Preserve unrelated hook entries, detect only explicit `llm-quota` ownership markers, and keep backup-before-change behavior. |
| T-02-05-02 | Tampering | `RunClaudeHookCacheWriter` cache output | mitigate | Validate both quota windows before writing and use temp-file-plus-rename via `writeJSONAtomic`. |
| T-02-05-03 | Information Disclosure | hook stdin parse errors | mitigate | Return concise errors without logging raw hook payload contents. |
| T-02-05-04 | Denial of Service | malformed hook stdin | mitigate | Fail the writer command without truncating existing cache because validation occurs before atomic rename. |
</threat_model>

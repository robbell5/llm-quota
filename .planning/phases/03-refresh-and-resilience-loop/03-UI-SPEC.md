# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Boundary

Phase 3 locks refresh and resilience behavior for the existing compact terminal UI.
It must not introduce final quota bars, threshold colors, polished stale/error copy,
or new narrow-pane row layouts; those remain Phase 4 responsibilities.

Visible rendering should remain aligned with the existing Phase 1/2 compact TUI:
title, divider, four placeholder-capable rows, and compact footer behavior around a
50-column tmux pane.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Terminal-equivalent two-space inline gaps between row fields |
| sm | 8px | Compact row grouping and footer separation where a blank line already exists |
| md | 16px | Existing shell horizontal padding total, represented as two terminal cells per side |
| lg | 24px | Reserved for later section padding; do not add in Phase 3 |
| xl | 32px | Reserved for later layout gaps; do not add in Phase 3 |
| 2xl | 48px | Reserved for major section breaks; do not add in Phase 3 |
| 3xl | 64px | Reserved for page-level spacing; do not add in Phase 3 |

Exceptions: Terminal cell widths are font-dependent. Keep existing constants
`shellHorizontalPadding = 4` and `shellPaddingX = 2`; do not add new padding,
margins, or row wrapping rules in Phase 3.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px terminal-default equivalent | 400 normal | 1.5 |
| Label | 14px terminal-default equivalent | 600 semibold via terminal bold | 1.5 |
| Heading | 16px equivalent, rendered as terminal-default bold title | 600 semibold via terminal bold | 1.2 |
| Display | 20px equivalent | 600 semibold | 1.2 |

Phase 3 implementation note: terminal UIs cannot reliably control pixel font
size. Preserve the current `LLM Quota` title and row labels; do not add new
typographic hierarchy beyond normal text and bold labels/title in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1e1e2e` Catppuccin Mocha base | App background and main surface |
| Secondary (30%) | `#313244` Catppuccin Mocha surface0 | Divider and low-emphasis structural line |
| Accent (10%) | `#89b4fa` Catppuccin Mocha blue | Title only in Phase 3 |
| Destructive | `#f38ba8` Catppuccin Mocha red | Destructive actions only; none exist in Phase 3 |

Accent reserved for: `LLM Quota` title only. Do not use accent for manual
refresh, transient refresh state, or broad interactivity in Phase 3.

Semantic warnings: `#f9e2af` Catppuccin Mocha yellow already exists for missing
placeholder glyphs. Phase 3 may set stale/error state internally, but final
visible stale warning colors and copy are deferred to Phase 4.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `r` key: Refresh data; do not add visible CTA text in Phase 3 |
| Empty state heading | Not displayed as a heading; keep four source rows in place |
| Empty state body | Existing row copy only: `— missing local data reset —` at wide row width, `— pending` at compact row width, or short label plus `—` below very narrow row width |
| Error state | No full-screen error. Preserve last-known-good rows; if no last-known-good rows exist, keep existing placeholders and typed source errors in model state for Phase 4 rendering |
| Destructive confirmation | None: no destructive actions in Phase 3 |

Forbidden visible copy in Phase 3: `refreshing...`, `last updated`, final stale
age text, final source-error hints, and new footer refresh hints unless existing
footer layout can include them without expanding Phase 4 display scope.

## Interaction Contract

| Interaction | Contract | Visible Feedback In Phase 3 |
|-------------|----------|-----------------------------|
| Initial launch | `Init` starts an immediate source refresh and schedules the 30-second tick | Existing rows remain stable; no separate loading banner |
| Automatic refresh | Refresh local Claude and Codex sources every 30 seconds | Successful data may update rows when render support exists; no visible tick indicator |
| Manual refresh | Pressing `r` triggers an immediate refresh | No visible `refreshing...` status; next scheduled tick remains on schedule |
| Duplicate manual refresh | Pressing `r` while refresh is already running coalesces or ignores the duplicate | No flicker, no duplicate source reads, no status copy |
| Resize | `tea.WindowSizeMsg` stores width and height, then rerenders current state | Must not trigger source reads or disturb the 30-second tick schedule |
| Source failure after success | Keep last-known-good windows independently per source and store typed error | Do not blank useful rows; Phase 4 owns final warning presentation |
| Initial source failure | Keep placeholder rows and hints; store typed error | No crash and no full-screen error |
| Stale source data | Treat captured data older than one hour as stale in model state/tests | Continue showing valid data; Phase 4 owns final stale styling/copy |

## State And Resilience Contract

| State | Contract | Source |
|-------|----------|--------|
| Freshness threshold | One hour for Claude and Codex | `03-CONTEXT.md` D-01, D-02 |
| Valid stale data | Show as valid data with warning state, not placeholders | `03-CONTEXT.md` D-03, D-04 |
| Last-known-good | Preserve independently for Claude and Codex | `03-CONTEXT.md` D-05 |
| Typed errors | Store missing, malformed, no-usable-event, and read errors in model state | `03-CONTEXT.md` D-07 |
| Refresh ownership | Startup, timer ticks, and manual `r` own data refresh | `03-CONTEXT.md` D-10, D-13, D-14 |
| Layout refresh | Resize rerenders only; no source reads | `03-CONTEXT.md` D-09, D-12 |

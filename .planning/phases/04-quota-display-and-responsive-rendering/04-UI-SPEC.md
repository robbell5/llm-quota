# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Boundary

Phase 4 delivers the final v1 quota dashboard rendering: four quota rows,
progress bars, threshold colors, reset countdowns, missing-source placeholders,
stale-source hints, and responsive row layouts for tmux-pane widths.

This phase must not add new data sources, network fallback, notifications,
history, forecasts, multi-account rendering, statusline integration, one-shot
output, or new keybindings. It renders existing source/model data only.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Terminal-equivalent single inline separator cell; shortest label/percent gaps |
| sm | 8px | Terminal-equivalent two-cell row field gaps and compact footer separation |
| md | 16px | Existing shell horizontal padding total; two terminal cells per side |
| lg | 24px | Header-to-row grouping equivalent; one divider line plus compact vertical rhythm |
| xl | 32px | Reserved; do not add larger dashboard gaps in Phase 4 |
| 2xl | 48px | Reserved; do not add major section breaks in Phase 4 |
| 3xl | 64px | Reserved; do not add page-level spacing in Phase 4 |

Exceptions: Terminal cells are not pixel-addressable. Preserve
`shellHorizontalPadding = 4` and `shellPaddingX = 2`. Row content must never
wrap; reduce label/reset copy and progress bar width before adding horizontal
space. No touch-target exception applies because this is keyboard-only TUI.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px terminal-default equivalent | 400 normal | 1.5 |
| Label | 14px terminal-default equivalent | 600 semibold via terminal bold | 1.5 |
| Heading | 16px equivalent, rendered as terminal-default bold title | 600 semibold via terminal bold | 1.2 |
| Display | 20px equivalent | 600 semibold | 1.2 |

Terminal UI implementation cannot reliably control pixel font size. Use only two
weights: normal text for percent/reset/footer/body and bold for `LLM Quota` plus
row labels. Do not add additional typographic hierarchy, all-caps warning text,
or blinking/emphasis variants.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1e1e2e` Catppuccin Mocha base | App background and dominant surface |
| Secondary (30%) | `#313244` Catppuccin Mocha surface0 | Divider, subtle progress-bar track, low-emphasis structure |
| Accent (10%) | `#89b4fa` Catppuccin Mocha blue | `LLM Quota` title only |
| Destructive | `#f38ba8` Catppuccin Mocha red | Destructive actions only; none exist in Phase 4 |

Accent reserved for: `LLM Quota` title only. Do not use blue for row urgency,
manual refresh, stale warnings, or broad interactivity.

Threshold semantic colors:

| State | Value | Rule | Usage |
|-------|-------|------|-------|
| Normal quota | `#a6e3a1` Catppuccin Mocha green | `used_percent < 60` | Progress fill and percent text |
| Elevated quota | `#f9e2af` Catppuccin Mocha yellow | `60 <= used_percent < 85` | Progress fill and percent text |
| High quota | `#f38ba8` Catppuccin Mocha red | `used_percent >= 85` | Progress fill and percent text only |

High usage must stay calm: no badges, alert icons, blinking, extra warning
words, or row-level background fills. Stale-but-valid rows keep threshold color
based on last-known percent; staleness is communicated through footer hints.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `r` key: Refresh data; do not add a visible CTA label unless it already fits in footer copy |
| Empty state heading | Not displayed as a heading; keep all four quota rows visible |
| Empty state body | Wide rows: `— missing local data reset —`; compact rows: `— pending`; very narrow rows: short label plus `—` |
| Error state | No full-screen error. If no last-known-good rows exist, show placeholder rows and a footer recovery hint: `Claude: run install-claude-hook`, `Claude: open Claude`, or `Codex: open Codex` |
| Destructive confirmation | None: no destructive actions in Phase 4 |

Footer copy must prioritize the most actionable next step and must fit the
current width. Do not expose internal source categories such as `malformed`,
`read_error`, or `no_usable_event` in user-facing copy.

Required footer hint priority:

1. Missing Claude data: `Claude: run install-claude-hook` if setup is needed;
   otherwise `Claude: open Claude`.
2. Missing Codex data: `Codex: open Codex`.
3. Stale Claude data: `Claude data 2h old; open Claude` using the largest useful
   age unit that fits.
4. Stale Codex data: `Codex data 2h old; open Codex` using the largest useful
   age unit that fits.
5. Baseline footer: `q / Ctrl-C quit`; include `r refresh` only when it fits
   without displacing recovery/setup hints.

If multiple hints apply, show the most actionable hint first, then only include
additional hints if the line remains within width. Do not rotate hints.

## Row Layout Contract

| Width | Required Content | Labels | Progress Bar | Reset Copy |
|-------|------------------|--------|--------------|------------|
| `>= 50` columns | Product/window label, progress bar, percent, reset token | Full labels: `Claude 5h`, `Claude 7d`, `Codex 5h`, `Codex 7d` | Visible, static, width-adjusted Bubbles progress track | `2h 14m` under 24h; `4d 06h` at or above 24h |
| `30-49` columns | Shortened label, progress bar if useful, percent, reset token | Abbreviate before removing bar: `Cl 5h`, `Cl 7d`, `Cx 5h`, `Cx 7d` | Keep while it can render as a meaningful visual cue | Use shortest non-wrapping reset token, still preferring two-part time |
| `< 30` columns | Short label, percent, reset token when it fits | Short labels only | Omit bars | Keep reset token if it fits; otherwise label plus percent is minimum |

Every rendered line, after ANSI stripping, must be `<= width` for widths 50, 49,
30, 29, and narrower defensive cases. Shrinking order is: footer copy, reset
prefix words, labels, progress bar width, then progress bar omission.

## Progress Bar Contract

Use `charm.land/bubbles/v2/progress` with static `ViewAs` rendering. Do not use
animated progress commands for quota bars.

| Property | Contract |
|----------|----------|
| Fill color | Green/yellow/red threshold color from `used_percent` |
| Percent text | Same threshold color as the bar fill |
| Track color | `#313244` Catppuccin Mocha surface0 |
| Width | Computed from available row width; never causes wrapping |
| Empty/missing data | Do not show an invented zero-value bar; show placeholders instead |
| Stale data | Render the last-known value normally, including threshold color |

---

## Reset Timing Contract

| Condition | Copy |
|-----------|------|
| Remaining time under 24 hours | `Xh Ym`, for example `2h 14m` or `0h 47m` |
| Remaining time at least 24 hours | `Xd YYh`, for example `4d 06h` |
| Reset time is now or in the past | `now` |
| Reset time unavailable | `—` |

Do not round multi-hour values up to a single coarse unit in the normal target
layout. Preserve the two-part format from the design spec unless the current
width requires a narrower form to avoid wrapping.

## Interaction Contract

| Interaction | Contract | Visible Feedback |
|-------------|----------|------------------|
| Initial launch | Render all four rows immediately, using placeholders until source data arrives | Stable rows, no full-screen loading state |
| Automatic refresh | Existing 30-second refresh cadence updates quota values | Rows update in place; no animation or tick indicator |
| Manual refresh | Pressing `r` triggers immediate refresh without disrupting scheduled ticks | No required transient status; footer may include `r refresh` only if it fits |
| Resize | `tea.WindowSizeMsg` rerenders current model state | Layout adapts without wrapping; no source refresh caused by resize |
| Quit | `q` and `Ctrl-C` exit cleanly | Footer may include `q / Ctrl-C quit` when it fits |
| High quota | Threshold styling changes at 60% and 85% | Color only; no alert language |
| Source failure with last-known-good data | Keep rendering last-known-good rows independently per source | Footer stale/recovery hint, rows remain data rows |
| Source failure without last-known-good data | Render placeholders for that source only | Footer recovery/setup hint; no crash |

## State And Resilience Contract

| State | Contract | Source |
|-------|----------|--------|
| Four-row order | Claude 5h, Claude 7d, Codex 5h, Codex 7d | `04-CONTEXT.md`, design spec |

| Missing source data | Placeholder rows plus actionable footer hint | `04-CONTEXT.md` D-09, D-10 |
| Multiple hints | Most actionable hint first; bounded to width; no rotation | `04-CONTEXT.md` D-12 |
| Source isolation | Claude errors do not blank Codex rows; Codex errors do not blank Claude rows | Phase 3 decisions |
| Test assertions | Strip ANSI and assert content, width, thresholds, missing/stale, narrow layouts | `04-CONTEXT.md` D-16 |

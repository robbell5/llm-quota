## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline visual separation; maps to the smallest terminal-cell gap when possible |
| sm | 8px | Compact row text gaps; maps to one terminal column gap where cell layout is required |
| md | 16px | Default horizontal grouping gap; maps to two terminal columns in normal rows |
| lg | 24px | Section separation; maps to one blank terminal row between row group and footer |
| xl | 32px | Reserved for wider pane affordances only; do not add new large gaps in Phase 7 |
| 2xl | 48px | Not used in Phase 7 |
| 3xl | 64px | Not used in Phase 7 |

Exceptions: Terminal rendering is cell-based. Preserve existing shell padding of 1 row top/bottom and 2 columns left/right. Quota rows use fixed terminal-cell columns, not pixel spacing: full label column 9 cells, compact label column 5 cells, percent column 4 cells, reset column 7 cells in normal rows, and a minimum progress width of 6 cells when bars are visible.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `r refresh` |
| Empty state heading | none - empty state is row-level, not a panel |
| Empty state body | Normal width row uses `missing local data`; compact row uses `pending`; very narrow row uses `—` only |
| Error state | Source freshness suffix uses `refresh failed`; first-run missing data footer remains `Claude: run install-claude-hook`, `Claude: open Claude`, or `Codex: open Codex` |
| Destructive confirmation | none |

Additional copy rules:

- The new Claude row label is `Sonnet 7d`.
- Claude row order is `Claude 5h`, `Claude 7d`, `Sonnet 7d`.
- Codex row order remains `Codex 5h`, `Codex 7d`.
- Normal freshness lines use `Claude updated 2:14 PM` and `Codex updated 2:14 PM`.
- When width allows, freshness lines include relative age: `Claude updated 2:14 PM (3m ago)`.
- Compact freshness lines keep source identity: `Claude 2:14 PM`.
- Very narrow freshness lines may shorten provider names: `Cl 2:14` and `Cx 2:14`.
- If stale age and current refresh failure both apply, use parenthetical status when width allows: `Claude updated 2:14 PM (2h old, refresh failed)`.
- Do not expose raw source error categories such as `malformed`, `read_error`, or `no_usable_event`.

## Phase 7 Row Contract

Normal layout applies when the inner content width is at least 46 cells. It must render a fixed right-side text region so percent and reset values line up across rows, including mixed values such as `0h 54m`, `21h 1m`, `100%`, `4d 06h`, `now`, and `—`.

Normal row columns:

| Column | Width | Contract |
|--------|-------|----------|
| Label | 9 cells | Full labels: `Claude 5h`, `Claude 7d`, `Sonnet 7d`, `Codex 5h`, `Codex 7d` |
| Gap | 2 cells | Stable separator |
| Progress bar | flexible, minimum 6 cells | Visible until it would crowd text columns |
| Gap | 2 cells | Stable separator |
| Percent | 4 cells | Right-align rendered percent or placeholder |
| Gap | 2 cells | Stable separator |
| Reset | 7 cells | Right-align reset countdown or placeholder |

Compact layout applies below normal width and at or above 26 cells. It must preserve label, percent, and as much reset context as fits. Compact labels may be `Cl 5h`, `Cl 7d`, `Sn 7d`, `Cx 5h`, and `Cx 7d`. Compact reset text should drop minutes and keep hours when needed for fit.

Very narrow layout applies below 26 cells. It must drop bars first, then reset text if needed. The minimum readable row is short label plus percent or placeholder. Do not wrap.

The Sonnet placeholder row is always present when Claude rows are visible. Missing Sonnet data must not remove the row. It uses the same columns as real rows with placeholder bar/marker, percent placeholder, and reset placeholder.

## Source Freshness Contract

Render one source freshness line immediately after each provider group:

1. Claude rows: `Claude 5h`, `Claude 7d`, `Sonnet 7d`, then Claude freshness.
2. Codex rows: `Codex 5h`, `Codex 7d`, then Codex freshness.

Freshness text is source-level, not row-level. Use the source windows' `CapturedAt`, `Stale`, and `StaleAge` values, falling back to `now - CapturedAt` when stale age is not already set.

Current refresh failures with last-known-good rows must appear on the affected freshness line, not only in the footer. Footer recovery hints remain for first-run missing data when a source has no rows.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| `r` | Request an immediate refresh. Do not reset the periodic 30-second tick. If a refresh is already in flight, coalesce the request. |
| `q` | Quit the TUI. |
| `Ctrl-C` | Quit the TUI. |
| Resize | Recompute row/freshness layout using the new width. No row or freshness line may exceed the terminal width. |
| Automatic refresh | Continue every 30 seconds. Preserve last-known-good rows when a source fails. |

No new keybindings, menus, dialogs, prompts, or destructive actions are introduced in Phase 7.

## Responsive Acceptance Targets

The implementation must be readable and line-width-safe at these tested terminal widths: 80, 50, 49, 30, 29, and 20 columns.

Required behavior:

- At 50 columns, quota rows keep bars, percent, reset text, and fixed right-column alignment.
- At 49 columns, labels may abbreviate, but percent and reset text stay readable.
- At 30 columns, bars may shrink or disappear; percent remains visible.
- At 29 columns, bars should be omitted if they would crowd text.
- At 20 columns, show the shortest readable label plus percent or placeholder; omit reset and bars when needed.
- All rendered lines, after ANSI stripping, must be less than or equal to the terminal width.
